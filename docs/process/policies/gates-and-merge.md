# Gate 与合并政策

## Gate 分层

| Gate | 用途 |
|---|---|
| G0 | 状态、文档、分支、范围、Secret 和 Workflow 静态安全 |
| G1 | 当前工作包的定向 Ruff、pytest 或 verifier |
| G2 | 仓库现有完整 `Test` 回归 |
| G3 | 真实但有限的模块/市场薄切片 |
| G4 | 产品级 E2E、多样本和性能矩阵 |
| G5 | 合并后独立验证 |

## 原则

- Codex 只运行 G1；G2–G5 在 Codex 会话之外执行。
- Gate 命令必须来自受信任的 `gate_profile`，不得从任务文本执行任意 Shell。
- 机械格式问题应由同版本工具修复并留下回归，不反复人工猜测。
- 创建分支和合并前各执行一次并行开发审计：开放 PR 文件交集、ahead/behind、Workflow、共享入口、Schema 和注册表。
- 合并前同步最新 `main`，重新运行受影响 Gate。
- 合并必须校验期望 Head SHA，防止检查后分支移动。
- 合并前全部通过仍不等于完成；G5 有阻断权。
- G5 失败时建立独立热修复 PR，修复后从最新 `main` 重跑受影响门禁，不得复用旧 PASS 宣称 100%。
