# 执行合同政策

## 开始条件

复杂任务开始前必须建立版本化任务合同，至少包含：目标、范围、输入、输出、允许/禁止来源、约束、验收标准、人工介入条件、回滚方式和通知策略。

## 角色

- **ChatGPT Web Supervisor**：理解需求、审计仓库、拆分工作包、诊断失败、创建和审查 PR、决定合并与人工决策。
- **GitHub Actions Executor**：环境预检、确定性命令、门禁、检查点、Artifact、通知和 post-merge。
- **Codex Thin Worker**：仅修改任务声明允许的路径，运行一个可信 Gate Profile，输出最小结构化结果。
- **用户**：发起任务并处理真正的权限、安全、不可逆或业务口径决策。

## 连续执行

正常路径必须连续推进。以下情况不得暂停：中间 Gate 通过、缓存命中、普通警告、可重试网络错误、阶段自然结束、覆盖缺口但程序执行正常。

只在以下情况暂停：

- 权限、登录、验证码或必要 Secret 缺失；
- 安全审计失败或 Codex 修改越界；
- 不可自动解决的代码冲突；
- 需要改变业务口径、来源优先级或数据语义；
- 破坏性或不可逆操作；
- 同一根因达到有限重试上限；
- 继续只能依靠猜测。

暂停必须给出已完成内容、根因、已尝试动作、最小人工动作、影响范围和恢复入口。
