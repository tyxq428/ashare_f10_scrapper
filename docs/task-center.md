# 任务中心：重试、版本与删除

任务中心将每次股票拉取保存为独立版本，并把固定接口清单中的每个请求组保存为可查看、可单独重试的子任务。

## 任务状态

- `PENDING`：等待执行。
- `RUNNING`：正在执行全部请求组。
- `RETRYING`：正在重新执行一个或多个子任务。
- `PARTIAL`：主流程已结束，但仍有失败子任务。
- `COMPLETED`：全部请求组成功，失败数为0。
- `FAILED`：任务未能生成有效数据包。
- `CANCELLED`：用户取消。

只有 `COMPLETED` 且失败数为0的任务会自动成为该股票的当前搜索版本。搜索、TTM、公式和按股票代码下载均读取 `data/<股票代码>/latest.json` 指向的任务。

## 子任务操作

展开任务卡后可以查看全部请求组的：

- 页面区块、接口家族和请求策略；
- 状态、记录数、执行次数和耗时；
- 是否使用精确回退；
- 错误信息和来源URL。

每个子任务都可以点击“重新执行”。该操作只清理并重新请求对应请求组的缓存，不会重跑已经成功的其他请求组。任务卡上的“重试失败项”会并行重新执行所有失败子任务，完成后集中重建JSON、Excel、Parquet和DuckDB。

## 重新执行全部

“重新执行全部”会创建一个新的任务ID和独立文件夹，保留原任务，便于版本比较和追溯。

## 当前数据版本

完整历史任务可以点击“设为当前版本”。当前版本在任务卡上显示 `CURRENT`，搜索页面也会显示正在使用的任务ID。

若删除当前版本，系统会自动选择同一股票最近一个失败0的完整历史任务；不存在完整历史版本时会删除 `latest.json`，搜索页面提示没有可用数据版本。

## 删除任务

删除操作会同时删除：

- SQLite中的任务与子任务记录；
- `data/<股票代码>/<任务ID>/` 整个文件夹；
- 该任务的原始缓存、检查点、JSON、Excel、Parquet和DuckDB。

运行中或重试中的任务不能删除。页面要求输入 `DELETE` 二次确认，并对删除路径执行严格校验，防止删除数据目录之外的文件。
