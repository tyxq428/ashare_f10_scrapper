# W01计划：官方PDF解析正确性闸门

## 目标

消除“附注编号被识别为金额”等解析假阳性，并建立可疑事实隔离机制，使低质量解析不会进入双源准确率、规范事实或研究映射包。

## 基线问题

真实样本中原始行：

```text
递延所得税资产 七、29
递延所得税负债 七、29
```

被解析为29元高置信官方事实。该错误不会形成双源冲突，因此必须在官方事实进入对账前独立拦截。

## 薄切片

覆盖五类情况：

1. 正常金额行；
2. 只有附注号、没有金额的行；
3. 附注号＋两个真实金额的行；
4. 括号负数和单位换算；
5. 无法确定时进入`PARSE_SUSPECT`，而不是补数。

## 实施内容

1. 新增官方事实质量闸门模块；
2. 为`OfficialFact`增加向后兼容的质量元数据；
3. 识别纯附注编号、页码、表序号和异常小整数；
4. 将可疑事实标记为`PARSE_SUSPECT`并从默认对账事实集中隔离；
5. 在官方验证和完整交叉验证两条路径中统一调用质量闸门；
6. 输出通过事实、可疑事实和质量统计；
7. 添加单元测试和真实问题回归测试。

## 验收标准

- `七、29`不再生成29元高置信事实；
- 正常报表金额不受影响；
- 可疑事实有原因、原始行、页码和文档信息；
- 可疑事实不计入比较准确率；
- 现有OfficialFact构造调用保持兼容；
- Ruff和相关Pytest通过。
