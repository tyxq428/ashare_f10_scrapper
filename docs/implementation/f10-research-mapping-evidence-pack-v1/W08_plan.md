# W08计划：多样本E2E、兼容性与性能验收

## 目标

在W01—W07数据契约稳定后，用真实双交易所样本和确定性边界样本验证正确性、兼容性、幂等、可恢复、可移动和资源边界，并形成可重复执行的验收脚本与CI矩阵。

## 样本矩阵

| 样本 | 验证目标 |
|---|---|
| 688521 | 上交所/科创板金标准、复杂PDF、解析质量闸门 |
| 002352 | 深交所/CNINFO、核心财务、资本事项与旧结果兼容 |
| 更正版本fixture | 截止日前后版本选择和边界文档 |
| 明确零fixture | `FACT_ZERO_EXPLICIT`与缺失不补零 |
| 制造业fixture | 存货、研发、分部和资本结构专题 |
| 北交所fixture | 官方来源未接入时的部分完成状态 |

## 实施内容

1. 新增确定性多样本验收脚本；
2. 新增Research Pack重导入和包移动验证；
3. 新增输入不变的缓存命中与稳定ID验证；
4. 新增配置/本体变化触发缓存失效的测试；
5. 新增原命令、旧导出和Raw Pack兼容性测试；
6. 新增首次运行、增量运行、表规模和文件规模指标；
7. 新增GitHub Actions多样本矩阵：
   - 确定性fixture矩阵；
   - 002352真实CNINFO薄切片；
   - Research Pack生成与质量验证；
8. 输出机器可读性能与兼容性报告；
9. 记录网络临时错误、重试次数和最终状态；
10. 所有失败均按数据、逻辑、来源、环境或权限分类。

## 验收标准

- Test、Ruff和编译检查通过；
- 688521现有四条核心工作流继续通过；
- 002352真实E2E通过且无未解释真实冲突；
- 更正版本和截止日回放无未来信息泄漏；
- 相同输入重复运行稳定且命中缓存；
- 包移动后JSON、Excel、Parquet和DuckDB仍可读取；
- 旧`fetch`、`run-and-validate`、`raw-pack`和现有API不破坏；
- 性能报告包含阶段耗时、缓存率、表行数和交付文件大小；
- 不需要人工介入即可进入W09。
