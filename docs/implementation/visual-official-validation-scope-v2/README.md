# 可视化全历史官方验证与执行中心UX优化

## 任务目标

将可视化执行中心从“需要用户理解内部技术参数”升级为“只需选择研究深度即可执行”的产品化入口，并让官方交叉验证默认支持自动识别上市日期和全部可用报告期。

## 任务合同

- **目标**：消除手工填写年度报告年份和一季度报告年份，默认一次验证上市以来所有官方定期报告；同时修复页面中其他容易误解或产生无效操作的交互。
- **输入**：当前`main`中的可视化执行中心、Raw Pack、全历史`FullCrossValidationRunner`、任务管理器和GitHub Actions。
- **输出**：简化后的网页、全历史官方验证后端集成、兼容旧任务的状态展示、测试、问题复盘与Windows启动说明。
- **并行开发约束**：不修改PR #21正在开发的`cross_validation/runner.py`、`validation/runner.py`和研究映射核心；只通过稳定公开接口`run_full_cross_validation`接入。
- **禁止事项**：不得要求用户先查询上市日期；不得把`NO_MATCH`或报告未披露解释为0；不得为了页面显示PASS而隐藏真实来源冲突；不得覆盖并行分支。
- **人工介入条件**：仅在GitHub权限阻止工作流写入、产生不可自动解决的代码冲突或需要改变业务口径时暂停。

## 完成标准

1. 网页不再显示“年度报告年份/一季度报告年份”。
2. 官方验证默认范围为“上市以来全部报告”，后台自动识别上市日期和实际报告期。
3. 支持“最近2个报告期、最近3年、最近5年、上市以来”四档范围。
4. 常用页面只保留股票代码、研究模式、官方验证范围和开始按钮；并发、重试、轮询、Raw Pack上限移入折叠的高级设置。
5. 旧任务不再显示`F10 UNKNOWN`；用户可见状态使用中文。
6. 下载区只展示真实可下载且有明确标签的Artifact，不再显示`fact_count`等内部键。
7. 全历史来源冲突显示为“完成但需复核”，不误写为执行失败。
8. Test、Visual Execution Control、E2E 688521、Raw Pack E2E、全历史官方验证集成测试全部通过。
9. 计划、每一步结果、检查点和问题复盘均保存于仓库。
