## 完成内容

- 核对了正式网页、前端配置提交、可视化API、任务Sidecar、Raw Pack和官方验证执行链。
- 确认旧网页将“两份报告薄切片”的`年度报告年份/一季度报告年份`直接暴露给最终用户。
- 确认仓库已有成熟的`run_full_cross_validation`全历史验证能力，可自动读取F10报告期、识别上市日期并下载全部可用官方报告。
- 识别并登记10项UX/执行风险，包括技术参数默认展开、示例股票误运行、旧任务`UNKNOWN`、内部字段错误下载链接、研究冲突与程序失败混用等。

- 在开始和合并前两个时点执行并行开发审计。

## 结论

正式用户不需要知道股票上市时间、最新年报年份或一季报年份。页面只应让用户选择研究深度与官方验证范围，报告期和上市日期由后台自动推导。
