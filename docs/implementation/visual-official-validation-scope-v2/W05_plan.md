# W05：合并、Post-merge验证与复盘

## 执行

1. 将功能分支同步到最新`main`。
2. 再次检查开放PR文件交集。
3. 合并前运行全部质量门禁。
4. 创建PR并记录变更、兼容策略和验证结果。
5. 在无重叠冲突时合并到`main`。
6. 通过临时验证分支触发合并后Test、Visual、E2E、Raw Pack和官方全历史验证。
7. 关闭临时PR并重置临时分支。
8. 更新集中问题与经验记录。

## 完成标准

- `main`包含新的自动范围UI和全历史官方验证调用；
- post-merge全部门禁通过；
- 临时文件和临时PR已清理；
- 剩余必做项为0；
- Windows正式启动入口仍为`main`和`/run.html`。
