# 验收矩阵与金标准样本

> 本文规定何时可以认为“F10研究映射包＋官方原文证据包”达到可交付状态。  
> 验收不以输出文件数量或字段分类100%为唯一标准，而以正确性、覆盖、来源、时点、可恢复和可消费性共同判断。

## 1. 总体完成定义

最终状态必须同时满足：

1. P0正确性问题全部关闭；
2. 规范事实、证据、血缘和状态模型稳定；
3. 688521和002352真实E2E通过；
4. 截止日回放无未来信息泄漏；
5. 覆盖与准确性指标分离；
6. 所有关键事实有证据或明确缺口状态；
7. 输出可移动、可重新导入、可增量更新；
8. 中断后可从检查点继续；
9. 不破坏现有F10默认命令和主要输出；
10. 最终文档、脚本、测试、日志和恢复说明齐全。

## 2. 验收状态

```text
PASS
PASS_WITH_NONBLOCKING_GAPS
PARTIAL_SOURCE_UNAVAILABLE
FAIL_DATA_CORRECTNESS
FAIL_SOURCE_CONFLICT
FAIL_POINT_IN_TIME
FAIL_SCHEMA
FAIL_REGRESSION
FAIL_RECOVERY
```

不得用`PASS_WITH_COVERAGE_GAPS`掩盖真实解析错误。只要存在未隔离的伪事实，必须失败。

## 3. 结构验收

| ID | 验收项 | 标准 | 失败示例 |
|---|---|---|---|
| S01 | 包目录完整 | manifest、schema、数据、质量、日志均存在 | 只有Excel，没有manifest |
| S02 | Schema可验证 | 所有JSON/Parquet/DuckDB符合版本化schema | 字段类型随运行漂移 |
| S03 | 唯一键 | source_fact_id、evidence_id、observation_id无重复 | 同一事实ID重复 |
| S04 | 文件可打开 | Excel/Parquet/DuckDB/JSON可重新读取 | XLSX损坏或只写A列 |
| S05 | 路径便携 | 项目包移动后相对证据路径仍有效 | 路径绑定临时运行目录 |
| S06 | 输出确定性 | 相同输入和版本重复运行产生同ID和稳定排序 | 每次事实ID变化 |
| S07 | 原始数据保留 | 规范化不覆盖原始响应或原文 | 只保留清洗结果 |
| S08 | 版本信息 | manifest记录代码、parser、registry、ontology版本 | 无法重放结果 |

| ID | 验收项 | 标准 |
|---|---|---|
| D01 | 附注号排除 | “七、29”等附注编号不得作为金额 |

| D06 | 负数 | 括号负数和报表绝对展示项目按指标规则处理 |

## 5. 来源和证据验收

| ID | 验收项 | 标准 |
|---|---|---|
| E01 | 文档身份 | document_id、标题、URL、SHA-256、发布机构完整 |
| E02 | 时间 | published_at、available_at、report_date、extracted_at完整 |
| E03 | 版本链 | 原版/更正版/撤回版关系可追溯 |
| E04 | 原文位置 | 直接事实有页码和行/表格/区域定位 |
| E05 | 原始文本 | evidence保存原始文本和text hash |
| E06 | 解析信息 | parser版本、方法、目标注册表hash存在 |
| E07 | 置信度 | 置信度由证据质量规则生成，不全为high |
| E08 | 可疑解析 | PARSE_SUSPECT不进入首选规范事实 |
| E09 | 来源等级 | 法定、政府、公司、平台、二手来源明确分层 |
| E10 | 证据完整率 | 核心官方直接事实100%；普通事实达到目标阈值 |
| E11 | 文件复用 | Raw Pack和官方验证对同一文件只下载/解析一次 |
| E12 | 权限状态 | 验证码/登录/403明确记录，不反复无效重试 |

## 6. Point-in-time验收

| ID | 场景 | 预期结果 |
|---|---|---|
| P01 | 截止日在原报告发布前 | 该报告不可进入基线 |
| P02 | 截止日在原版发布后、更正版发布前 | 使用原版 |
| P03 | 截止日在更正版发布后 | 使用更正版，保留原版版本链 |
| P04 | 运行日在截止日之后 | 不得因当前查询结果回填未来文件 |
| P05 | 未来事件仅作边界披露 | 状态为BOUNDARY_DISCLOSURE，不参与历史模型 |
| P06 | 同一as_of_date重复运行 | 规范事实集合稳定 |
| P07 | 不同as_of_date对比 | 输出明确事实新增、修订和撤回diff |

任何未来信息泄漏均为`FAIL_POINT_IN_TIME`。

## 7. 对账和指标验收

## 7.1 容差

不同指标必须采用不同策略：

| 类型 | 示例 | 比较方式 |
|---|---|---|
| 大额金额 | 总资产、收入 | 规范元单位＋绝对/相对容差 |
| 每股数据 | EPS、每股净资产 | 小数位和相对容差 |
| 百分比 | 毛利率、持股比例 | 基点容差 |
| 股数/人数 | 总股本、员工数 | 整数或明确单位容差 |
| 日期 | 公告日、解禁日 | 日期精确匹配或规则化集合 |
| 文本 | 审计意见、报告类型 | 规范文本/枚举匹配 |
| 集合 | 十大股东、董事名单 | 集合匹配并保留差异 |

## 7.2 质量指标

必须分别输出：

| 指标 | 公式 | 解释 |
|---|---|---|
| classification_coverage | 已分类字段上下文/全部上下文 | 注册表完整性 |
| report_discovery_coverage | 已发现目标报告/目标报告 | 来源发现能力 |
| document_download_rate | 成功下载/已发现 | 下载能力 |
| document_parse_rate | 成功解析/已下载 | 文档解析能力 |
| target_extraction_coverage | 已提取目标事实/预期目标事实 | 解析覆盖 |
| comparison_coverage | 两源都有值/目标源事实 | 可比覆盖 |
| comparison_accuracy | 匹配/(匹配+真实冲突) | 已比较事实准确性 |
| evidence_completeness | 完整证据事实/官方事实 | 来源穿透 |
| suspicious_extraction_rate | 可疑解析/全部候选解析 | 解析风险 |
| unresolved_rate | 未解决/目标任务单元 | 剩余缺口 |

不得再将MISSING_OFFICIAL大量计入一个名为“准确率”的指标。

## 10. 金标准回归案例

## 10.1 688521：解析正确性金标准

### 必测事实

- 资产总计；
- 负债合计；
- 所有者权益；
- 营业收入；
- 归母净利润；
- 扣非归母净利润；
- 经营现金流；
- 研发费用；
- 递延所得税资产/负债所在跨行表格。

### 必测错误

```text
原始行：递延所得税资产 七、29
错误旧结果：29元
新预期：不得生成29元事实；正确金额可重建则提取，否则标记解析覆盖缺口。
```

### 必测范围

- 2025年报；
- 2026Q1；
- 历史报告覆盖在PR #18合并后补充；
- 更正版本优先；
- 页面和原始行完整。

## 10.2 002352：深市/CNINFO回归

必须保持：

- 官方报告发现；
- 两期核心财务匹配；
- 会计逻辑6/6；
- TTM双公式；
- A/H或资本事项不被误当成定期财务事实；
- 旧已验证结果不得因本体重构出现无解释退化。

## 10.3 更正报告样本

需要验证：

- 原版与更正版都缓存；
- as-of-date决定采用版本；
- fact diff准确；
- 旧研究快照可重放。

## 10.4 明确零样本

需要验证：

- 报告明确“0”或“无” -> FACT_ZERO_EXPLICIT；
- 空白、横线或未检索到不得自动视为零；
- 证据保存原文语境。

## 10.5 事件样本

至少包括：

- 回购；
- 股权激励；
- 可转债；
- 重大合同或重组；
- 一个监管/诉讼事件。

验证事件唯一键、公告版本、可得日和与规范事实的映射。

## 11. 逻辑验收

至少全量运行：

- 资产=负债+权益；
- 负债和权益合计=资产；
- 净利润=归母+少数；
- 现金净增加额关系；
- CFO/CFI/CFF与现金变动；
- FY与四个独立季度；
- TTM两种公式；
- 分部收入与合并收入；
- 股本桥；
- 同比/环比期间一致性；
- 公式输入无循环和跨版本污染。

逻辑失败不得被“数值双源匹配”覆盖。

## 12. 缓存、恢复和性能验收

| ID | 验收项 | 标准 |
|---|---|---|
| C01 | 原始缓存命中 | 未变化文档不重复下载 |
| C02 | 解析缓存命中 | 版本相同不重复解析 |
| C03 | 配置失效 | registry/ontology变化时相关缓存失效 |
| C04 | 检查点 | 每个工作包有可恢复状态 |
| C05 | 重试队列 | 临时失败可增量重试 |
| C06 | 幂等 | 重复执行结果稳定 |
| C07 | 批量写入 | 并发读取、集中写入，避免同文件竞争 |
| C08 | 性能报告 | 记录下载、解析、映射、比较、导出耗时和缓存率 |
| C09 | 异常恢复 | 中断后不重跑已成功文档/事实 |
| C10 | 资源边界 | 大历史包不会因单文件导致全部成果丢失 |

## 13. 文档验收

每个阶段必须存在：

- 阶段计划；
- 改动范围；
- 测试矩阵；
- 执行结果；
- 新增通过数；
- 失败与重试；
- 缓存命中率；
- 未解决项；
- 下一恢复入口；
- 影响和回滚说明。

所有文档提交到本规划目录或后续正式设计目录。

## 15. 合并闸门

只有同时满足以下条件才允许合并实现PR：

1. 688521附注号回归通过；
2. 002352回归无无解释退化；
3. 全部单元测试和ruff通过；
4. 真实E2E至少两个样本通过；
5. as-of-date泄漏测试通过；
6. 公式/逻辑错误为零；
7. 输出可重新导入；
8. 文档和迁移说明齐全；
9. 不与未合并并行PR产生未解决冲突；
10. 所有P0问题关闭，P1缺口有明确状态和后续计划。
