## 1. 最终目标

在现有 A 股 F10 平台、免费官方披露交叉验证和 Raw Pack 基础上，建设两项可跨公司复用、可增量更新、可穿透验证的标准产品：

1. **F10研究映射包（Research Mapping Pack）**
   - 将东方财富等源的宽表、长表和事件记录，转换为研究系统可以直接消费的规范化事实与研究模块输入；
   - 明确一项事实的指标身份、期间、口径、单位、来源、版本、可得日、验证状态和下游用途；
   - 将“源字段很多”升级为“研究字段可用、口径唯一、可追溯、可增量更新”。

2. **官方原文证据包（Official Evidence Pack）**
   - 自动发现、下载、缓存、解析和版本化保存法定披露及重要官方原文；
   - 每一项关键事实可穿透到具体文档、页码、表格/行、原始文本和文件哈希；
   - 能明确区分：官方直接披露、由官方事实计算、未加载文档、解析未覆盖、来源未披露、来源冲突和明确为零。

两者共同服务于后续财务模型、盈利质量、分部 KPI、股本桥、预期反推、估值和投资决策系统，但不把投资判断伪装成数据事实。

## 2. 任务范围

### 2.1 本轮纳入

- 审计现有 F10、官方验证、全字段交叉验证和 Raw Pack 的代码边界；
- 审计样本 `688521` 的两份最新输出：
  - `688521_official_full(1)(1).xlsx`；
  - `688521_cross_validation(1)(1).xlsx`；
- 修复官方 PDF 解析和事实生成中的系统性正确性问题；
- 建立研究级规范事实模型、证据模型、状态模型和版本模型；
- 统一 F10、官方报告、Raw Pack、交叉验证之间的数据血缘；
- 增加 point-in-time / as-of-date 控制；
- 建立更准确的覆盖率、匹配率和冲突率指标；
- 建立薄切片、全量 E2E、回归测试、检查点和恢复机制；
- 输出 JSON、Parquet、DuckDB、Excel 与可读证据索引；
- 每一阶段先提交对应 Markdown 规划/审计/结果文档，再连续执行代码任务。

### 2.2 本轮不纳入

- 不开发投资观点、目标价或仓位建议；
- 不将 Wind、Choice、iFinD 等收费源作为必需依赖；
- 不绕过登录、验证码、付费墙或网站权限控制；
- 不以 OCR 作为默认 PDF 解析路径；
- 不自动把“未找到”写成零或不存在；
- 不在没有验证的情况下扩大到全市场批量运行；
- 不与当前并行中的可视化执行中心 PR 混合开发；
- 不在本阶段修改任何代码。

## 4. 允许的数据源

按优先级：

1. 上交所、深交所、北交所、巨潮资讯及其正式披露文件；
2. 中国证监会、政府部门、司法机关和其他官方平台；
3. 上市公司官网、投资者关系网页和正式公告；
4. 东方财富公开 F10 接口，作为结构化候选事实与市场平台口径；
5. 其他免费公开来源，仅作为补充或线索，并明确来源等级。

## 5. 禁止事项

- 将附注编号、页码、项目编号误识别为金额；
- 将母公司口径与合并口径混用；
- 将累计季度与独立单季度混用；
- 将当前抓取到的未来信息回填至历史研究时点；
- 将市场价格派生指标误分类为法定报告直接事实；
- 将解析器未覆盖误写成公司未披露；
- 将官方报告存在但 F10 缺失的事实静默丢弃；
- 用一个总“匹配率”同时表达解析覆盖与数值准确性；
- 用未经验证的模型推断补齐事实空白；
- 在没有检查点的情况下执行长批次或高风险重构。

## 6. 目标输出

### 6.1 F10研究映射包

至少包含：

- `canonical_facts`：规范化事实长表；
- `source_facts`：保留所有原始源记录；
- `fact_lineage`：源事实到规范事实的多对一血缘；
- `financial_statements`：三张表规范事实；
- `quarterly_and_ttm`：累计、单季度和 TTM；
- `derived_metrics`：确定性公式事实；
- `segments_and_kpis`：分部与行业 KPI；
- `capital_structure`：股本、回购、激励、可转债、解禁等；
- `governance_and_events`：治理、资本事项和事件；
- `market_and_consensus`：价格、估值与预测，明确非官方属性；
- `coverage_and_gaps`：覆盖、缺口、冲突和待人工项。

### 6.2 官方原文证据包

- 页级索引；
- 事实级证据对象；
- 文档、页码、原始行、表格/区域标识；
- 解析器版本、目标注册表版本和本体版本；
- 未解析、可疑解析、来源冲突和权限阻塞队列。

## 7. 完成标准

### 7.1 正确性

- `688521` 样本中“递延所得税资产 七、29”等附注号不再生成金额事实；
- 所有新增规范事实均能追溯到源事实或明确公式；
- 高风险事实不存在未标记的单位、期间、范围或版本歧义；
- 真实公式和会计逻辑错误为零；
- 关键回归样本无未解释的真实冲突。

### 7.2 覆盖与可解释性

分别报告而不是混合：

- 字段分类覆盖率；
- 目标文档发现率；
- 文档下载率；
- 文档解析成功率；
- 官方目标字段提取覆盖率；
- 双源可比覆盖率；
- 双源数值匹配准确率；
- 官方仅有/F10 仅有比例；
- 证据完整率；
- 未解决率。

### 7.3 Point-in-time

- CLI 和核心 API 支持 `as_of_date`；
- 截止日后文档不得进入基线事实；
- 更正/修订版本必须形成版本链；
- 每项事实记录 `published_at`、`available_at`、`effective_at/report_date` 和 `extracted_at`。

### 7.4 工程质量

- 所有新增确定性逻辑有单元测试；
- 至少两个交易所样本通过 E2E；
- 缓存键包含文档哈希、解析器版本、目标注册表哈希和本体版本；
- 重复运行幂等；
- 中断后可从最近检查点恢复；
- 最终输出可重新导入并通过 schema 验证。

## 8. 优先指标

主指标：

```text
新增通过验收的研究级规范事实数 / 本轮处理的目标事实数
```

辅助指标：

- 可比事实准确率；
- 官方目标字段提取覆盖率；
- 证据完整率；
- 可疑解析召回率；
- 一次通过率；
- 返工率；
- 缓存命中率；
- 增量运行耗时；
- 人工介入率。

## 9. 人工介入规则

仅在以下情况暂停：

- 权威官方来源之间存在无法自动判定的版本/口径冲突；
- 需要用户决定研究口径，而该选择会改变规范事实定义；
- 需要登录、验证码或付费权限；
- 操作涉及不可逆数据删除或生产发布；
- 环境无法运行必要测试，且没有替代验证路径。

其他解析缺口、单个来源失败或非阻塞字段缺失进入明确状态和重试队列，不中止整体执行。
